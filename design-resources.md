# 设计资源库 - 提升 Vibecoding 设计感

> **适用场景**：在 AI 编程中需要设计参考、UI 截图、组件灵感时使用
> **兼容工具**：Cursor / Windsurf / Trae / GitHub Copilot / Claude Code / 任何支持 System Prompt 的 AI 编程工具
> **版本**：v1.0 | **更新日期**：2026-06-16

---

## 使用方式

### 触发词
`找参考`、`找设计灵感`、`vibecoding`、`设计参考`、`找截图`、`UI参考`、`找类似网站`

### 核心用法

将以下任一网站的截图/链接发给 AI，配合提示词：

```
请参考这个[网站/截图]的设计风格，帮我实现[具体功能/页面]。
关键要复刻：[配色/布局/交互/组件风格]
```

## 详细使用指南

### 1. Curated Design — curated.design
**精选网页设计灵感库**

- 收录顶级商业网站截图，覆盖各行业风格与布局
- **用法**：开启 vibe 前先找参考，直接甩给 AI「做成这个风格」，省去风格描述的麻烦
- **适合**：全站风格参考、配色方案提取、布局结构借鉴
- **分类**：按行业/风格/布局类型筛选

### 2. Landing Love — landing.love
**落地页设计案例集**

- 专注落地页结构与转化设计，提供真实项目截图与分析
- **用法**：快速拆解「Hero + 价值主张 + CTA」结构，让 AI 照着搭
- **适合**：Landing Page / 产品首页 / 营销页面
- **核心要素**：Hero 区、价值主张展示、社会证明、CTA 按钮

### 3. SaaSPO — saaspo.com
**SaaS 官网设计合集**

- 垂直于 SaaS 产品的官网截图库，聚焦 B2B/B2C 软件产品展示方式
- **用法**：做产品官网时找同类对标，告诉 AI「参考这种 SaaS 风格」
- **适合**：SaaS 官网 / 产品介绍页 / 定价页 / 功能展示页
- **节省**：大量 prompt 描述时间

### 4. Navbar Gallery — navbar.gallery
**导航栏设计灵感库**

- 专门收录各类导航栏设计，含布局、交互与视觉风格变体
- **用法**：导航是每个项目都要做的组件，直接选一个截图给 AI 复刻
- **适合**：顶部导航 / 侧边栏 / 移动端菜单 / 多级菜单
- **效果**：精准高效，避免从零设计

### 5. CTA Gallery — cta.gallery
**行动召唤区设计库**

- 汇聚各种 CTA 区块的设计范式，涵盖文案结构、按钮样式与背景处理
- **用法**：CTA 决定转化率，参考真实高转化案例指导 AI 生成
- **适合**：注册按钮 / 订阅区 / 下载引导 / 联系表单
- **效果**：比自己想文案和布局更快更好

### 6. Mobbin — mobbin.com
**移动端 UI 截图库**

- 收录主流 App 的完整界面截图，支持按场景和交互类型筛选
- **分类**：
  - 行业：Finance / Social / Lifestyle / Shopping / Portfolio
  - 页面：Hero / FAQ / Pricing / Blog / Footer
- **用法**：做移动端时找真实 App 界面参考，把截图给 AI 实现
- **效果**：避免输出「教科书式」假界面，输出真实可用的 UI

### 7. Rebrand Gallery — rebrand.gallery
**品牌重塑案例库**

- 展示知名品牌升级前后对比，聚焦 Logo、色板与视觉系统演变
- **用法**：建立品牌视觉语言时找参照，快速提炼风格关键词再传给 AI
- **适合**：品牌色板 / Logo 设计 / 视觉系统 / 品牌指南
- **效果**：输出结果更有品牌感

### 8. Huge Icons — hugeicons.com
**专业图标资源库**

- 提供 4000+ 高质量 outline / duotone 图标
- 支持多格式下载（SVG/PNG/JSON）与 API 接入
- **用法**：vibe coding 最怕图标风格乱，用统一图标库 + 直接引用名称让 AI 保持视觉一致性
- **适合**：所有项目统一图标来源
- **格式**：outline（线框）/ duotone（双色）/ solid（实心）三种风格

### 9. Component Gallery — component.gallery
**组件与设计系统合集**

- 汇聚主流设计系统的组件截图与实现对比
- 覆盖：按钮 / 表单 / 卡片 / 导航 / 表格 / 模态框等常用模式
- **用法**：让 AI 参考成熟设计系统的组件规范
- **效果**：生成代码更符合工程标准，减少返工

### 10. Pageflows — pageflows.com
**用户流程截图集**

- 收录各 App 的完整用户流程截图（注册→使用→付费全链路）
- 支持 iOS / Android / Web 三端筛选
- **用法**：设计用户流程时的完整参考
- **适合**：新用户引导 / 核心功能流程 / 支付流程 / Onboarding

## 高效提示词模板

### 模板1：风格复刻
```
我正在做[项目名]的[页面/功能]，请参考以下设计风格实现：

【参考来源】
- 网站：[网站名称] URL
- 截图：（粘贴截图或描述截图内容）

【我要复刻的元素】
- 配色：[主色/辅色/背景色]
- 布局：[整体结构]
- 组件风格：[按钮/卡片/字体等]
- 特殊效果：[动画/渐变/阴影等]

【技术约束】
- 技术栈：[React/Vue/其他]
- UI库：[shadcn/ui/Ant Design/其他]

请生成完整的[组件/页面]代码。
```

### 模板2：组件复刻
```
请参考 [navbar.gallery/cta.gallery/component.gallery] 中的[具体设计]，
帮我实现一个[组件名]。

要求：
- 视觉上尽量还原参考设计的风格
- 使用 [技术栈] + [UI库] 实现
- 包含 hover/focus/disabled 等状态
- 响应式适配
```

### 模板3：移动端复刻
```
我在做[App类型]的移动端应用，请参考 mobbin.com 中[行业]分类下的[页面类型]截图，
帮我实现[具体页面/功能]。

参考要点：
- 导航模式：[底部标签/侧滑/其他]
- 列表样式：[卡片/纯文字/图文混排]
- 交互细节：[下拉刷新/滑动操作/其他]
```

### 模板4：图标统一
```
本项目统一使用 hugeicons.com 的[outline/duotone]风格图标。

请为以下功能选择合适的图标名称：
- [功能1]：[图标名]
- [功能2]：[图标名]

图标引入方式：从 hugeicons.com 下载后放入 src/icons/
或使用 @hugeicons/react-pro 包直接引用
```

## Vibecoding 最佳实践

1. **先找参考再动手**：不要凭空描述设计，先从资源库找到接近的截图
2. **截图 > 文字描述**：一张截图胜过 500 字的风格描述
3. **明确告诉 AI 要复刻什么**：配色？布局？交互？组件？越具体越好
4. **一次只复刻一个区域**：不要试图一次复刻整个页面，分区域逐步实现
5. **保持资源一致性**：图标用一个库、配色从一个参考提取、组件风格统一
