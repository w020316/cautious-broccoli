# 提示词优化指南 - API 缓存命中率 + Token 消耗最低

> **适用场景**：优化 AI 编程工具的 API 调用效率，降低成本，提升响应速度
> **兼容工具**：Cursor / Windsurf / Trae / GitHub Copilot / Claude Code / 任何基于 LLM API 的编程工具
> **版本**：v2.0 | **更新日期**：2026-06-16

---

## 一、API 缓存命中率优化

### 1.1 缓存原理

主流 AI API 都支持**前缀缓存（Prefix Caching）**：对请求前缀缓存，后续请求前缀匹配时跳过重复计算。

| 平台 | 缓存类型 | 最小前缀 | 缓存时长 | 节省 |
|------|---------|---------|---------|------|
| Anthropic(Claude) | 自动前缀缓存 | 1024 tokens | 5分钟 | 输入-90% |
| OpenAI(GPT) | 自动前缀缓存 | 1024 tokens | 5-10分钟 | 输入-50% |
| Google(Gemini) | 上下文缓存 | 自定义 | 最长1小时 | 输入-75% |

### 1.2 核心策略

**策略1：System Prompt 按稳定性分层排列**

```
第一层：核心身份（永不变）→ "你是全栈开发专家团队..."
第二层：硬规则（极少变）→ 阶段隔离/文档驱动/测试先行
第三层：输出规范（极少变）→ 简洁/增量/表格
第四层：技术栈（偶尔变）→ 选型表/组件库/免费资源
第五层：流程（按阶段变）→ 当前阶段和任务
```

前4层占 System Prompt 的 80%+，在多轮对话中保持不变，缓存命中率极高。

**策略2：文档引用代替文档复制**

```
❌ [粘贴500行需求文档全文]                    → ~5000 tokens，每次重算
✅ 参考 specs/需求文档.md，本次涉及F001登录    → ~50 tokens，命中缓存
```

**策略3：统一 System Prompt 模板**

为项目创建固定模板，所有对话共用。只有"当前阶段"和"当前任务"变化：

```
# 身份 / 硬规则 / 输出规范 / 技术栈    ← 不变部分(80%+)
# 当前阶段：编码实现                     ← 偶尔变化
# 当前任务：实现用户登录                 ← 每次变化
```

**策略4：项目规则文件放静态内容**

各平台配置文件（.cursorrules / CLAUDE.md 等）中只放静态规则，不放动态内容：

```
# .cursorrules 示例（高缓存命中）
- 技术栈：React + TypeScript + Tailwind CSS
- 组件：函数组件 + Hooks，shadcn/ui
- 状态：Zustand
- API：src/api/ 统一封装
- 命名：camelCase变量/PascalCase组件/kebab-case文件
- 样式：Tailwind优先，不写内联样式
```

### 1.3 缓存命中率优化效果

| 优化措施 | 缓存命中率提升 | 成本节省 |
|---------|--------------|---------|
| 静态内容前置 | +40% | 输入Token费用-36% |
| 统一Prompt模板 | +30% | 输入Token费用-27% |
| 文档引用代替复制 | +20% | 输入Token费用-18% |
| 项目规则文件 | +15% | 输入Token费用-14% |
| **综合优化** | **+60%** | **输入Token费用-54%** |

## 二、Token 消耗最小化

### 2.1 核心原则

| 原则 | 说明 | 节省 |
|------|------|------|
| 精准指令 | 说清要什么，不说废话 | 30-50% |
| 结构化输出 | 表格/列表代替段落 | 20-40% |
| 增量变更 | 只输出变更部分 | 50-80% |
| 按需加载 | 只传必要上下文 | 30-60% |
| 分阶段执行 | 一次只做一个任务 | 20-40% |

### 2.2 精准指令

```
❌ (~200 tokens)：
你好，我想请你帮我做一个用户登录的功能。这个功能需要支持邮箱登录和微信登录，
邮箱登录需要验证码，微信登录需要扫码。登录成功后要跳转到首页，如果失败要显示
错误提示。另外还需要记住登录状态，下次打开不用重新登录。请帮我实现这个功能。

✅ (~60 tokens)：
实现用户登录：
- 邮箱(验证码) + 微信扫码
- 成功→首页，失败→错误提示
- 记住登录(7天)
参考：specs/features/登录/方案.md
```

### 2.3 增量变更

```
❌ 输出整个500行文件

✅ 只输出变更：
文件：src/components/LoginForm.tsx
变更：handleLogin 函数
- 添加微信登录分支
- 修改错误处理

[只输出变更的函数代码]
```

### 2.4 上下文管理

| 策略 | 操作 |
|------|------|
| 按需加载 | 只读必要文件，不全量读取 |
| 摘要代替全文 | "需求摘要：P0=登录/注册/首页，详见 specs/需求文档.md" |
| 新窗口清理 | 超10轮开新窗口，先保存 specs/进度.md |
| 对话压缩 | "总结进展保存到 specs/进度.md，我们开新窗口继续" |

### 2.5 对话压缩模板

```
请总结当前进展：
- 已完成：[列表]
- 当前任务：[描述]
- 待解决：[列表]
- 下一步：[描述]

保存到 specs/进度.md，然后我们开新窗口继续。
```

## 三、跨平台兼容性

### 3.1 通用提示词结构

以下结构在所有主流 AI 编程工具中都能正常工作：

```
# 角色
你是[角色]，负责[职责]。

# 硬规则
1. [规则1]
2. [规则2]

# 步骤
1. [步骤1]
2. [步骤2]

# 输出
[格式模板]
```

### 3.2 各平台配置文件

| 平台 | 配置文件 | 位置 | 格式 |
|------|---------|------|------|
| Cursor | `.cursorrules` | 项目根目录 | 纯文本 |
| Windsurf | `.windsurfrules` | 项目根目录 | 纯文本 |
| Trae | `.trae/rules/project_rules.md` | 项目目录 | Markdown |
| Claude Code | `CLAUDE.md` | 项目根目录 | Markdown |
| GitHub Copilot | `.github/copilot-instructions.md` | 项目根目录 | Markdown |
| Cline | `.clinerules` | 项目根目录 | 纯文本 |
| Aider | `.aider.conf.yml` | 项目根目录 | YAML |

### 3.3 平台特定优化

| 平台 | 优化技巧 |
|------|---------|
| Cursor | `@file`引用文件 / `.cursorrules`放静态规则 / Composer模式处理多文件 |
| Windsurf | Cascade模式多步骤 / `.windsurfrules`定义规则 / Memories保存上下文 |
| Trae | Skill系统触发工作流 / `.trae/rules/`定义规则 / 多窗口管理阶段 |
| Claude Code | `CLAUDE.md`定义规则 / `/init`初始化 / `@file`引用 |
| Copilot | `.github/copilot-instructions.md` / Copilot Chat中用触发词 |

### 3.4 统一项目规则模板

```
# 项目规则

## 技术栈
- 前端：[框架] + [语言] + [样式]
- 后端：[框架] + [语言]
- 数据库：[类型]
- 部署：[平台]

## 编码规范
- 命名：camelCase变量 / PascalCase组件 / kebab-case文件
- 样式：[方案]优先
- 状态：[方案]
- API：统一封装
- 错误：统一模式

## 开发流程
- 先读 specs/ 再动手
- 一次一个任务
- 完成写测试
- 按文档执行

## 输出要求
- 代码只输出变更部分
- 说明用表格
- 不输出未修改代码
- 不用过渡语
```

## 四、成本估算

### 4.1 优化前后对比

| 场景 | 未优化 | 优化后 | 节省 |
|------|--------|--------|------|
| 单次功能开发 | ~15K tokens | ~5K tokens | 67% |
| 完整项目(10功能) | ~200K tokens | ~60K tokens | 70% |
| 日常迭代(5次/天) | ~50K/天 | ~15K/天 | 70% |
| 月度成本(GPT-4o) | ~$300/月 | ~$90/月 | 70% |

### 4.2 v3.0 提示词 vs v2.0 提示词

| 指标 | v2.0 | v3.0 | 改进 |
|------|------|------|------|
| System Prompt tokens | ~1200 | ~800 | -33% |
| 阶段提示词平均tokens | ~400 | ~200 | -50% |
| 缓存命中率 | ~60% | ~80% | +20% |
| 单次对话总消耗 | ~8K | ~4K | -50% |

v3.0 优化要点：
- System Prompt 精简33%，去除冗余描述
- 阶段提示词压缩50%，用紧凑格式替代完整模板
- 输出模板用行内格式（`| 列1 | 列2 |`）替代多行展开
- 硬规则合并去重，5条→3条
